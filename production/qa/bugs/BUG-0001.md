# Bug Report

## Summary
**Title**: Arcade: il timer di intermission (4s) scade mentre la schermata di scelta modificatore è ancora aperta, causando sovrapposizione di stati e potenziale corruzione della successione eventi
**ID**: BUG-0001
**Severity**: S2-Major
**Priority**: P1-Immediate
**Status**: Open
**Reported**: 2026-05-04
**Reporter**: psychosocial

## Classification
- **Category**: Gameplay
- **System**: Boss defeat / Arcade modifier choice flow
- **Frequency**: Always (riproducibile ogni ciclo boss in Arcade)
- **Regression**: Unknown

## Environment
- **Build**: v7.20+
- **Platform**: All
- **Scene/Level**: Post-boss defeat, ciclo Arcade
- **Game State**: INTERMISSION che transita a PLAY con overlay modificatore ancora visibile

## Reproduction Steps
**Preconditions**: Arcade mode, raggiungere e sconfiggere un boss (es. C1 FED)

1. Sconfiggi il boss in Arcade mode
2. Dopo ~5 secondi parte l'intermission con timer 4.0s
3. Dopo ~1.5s dall'inizio intermission appare la schermata scelta modificatore (3 carte)
4. **Non scegliere subito** — attendi che il timer intermission scada (4.0s totali)
5. A 4.0s il WaveManager genera `START_WAVE`, il gioco transita a PLAY, i nemici spawnano
6. La schermata di scelta modificatore è ancora visibile sopra il gameplay

**Expected Result**: La schermata modificatore dovrebbe bloccare la progressione fino alla scelta. L'intermission dovrebbe durare fino a dopo la scelta.

**Actual Result**: Il timer intermission non è accoppiato alla scelta modificatore. Dopo 4 secondi il ciclo successivo parte automaticamente anche se il giocatore non ha ancora scelto.

## Technical Context

Il problema è in `src/core/GameplayCallbacks.js:532-554` (callback `onBossDeath`):

```js
// line 540: startIntermission viene chiamato subito (timer = 4.0s)
d.startIntermission(d.t('CYCLE') + ' ' + newCycle + ' ' + d.t('BEGINS'));
// line 541: dopo 1.5s mostra la schermata modificatore
d.bossDeathTimeout(() => {
    G.ModifierChoiceScreen.show(picks, () => { /* ... */ });
}, 1500);
```

Il `WaveManager.update()` decrementa `intermissionTimer` ogni frame e a 0 ritorna `START_WAVE` (`src/managers/WaveManager.js:57-61`). Questo avviene indipendentemente dallo stato della schermata modificatore.

Inoltre, `src/ui/ModifierChoiceScreen.js:98-100` prova a bloccare input durante INTERMISSION/PAUSE ma controlla `window.Game.gameState` che **non esiste** — `gameState` è una variabile locale di `main.js`, mai esposta come proprietà di `window.Game`. La guardia è sempre falsa.

- **Likely affected files**:
  - `src/core/GameplayCallbacks.js:532-554` (root cause)
  - `src/ui/ModifierChoiceScreen.js:96-100` (guardia input rotta)
  - `src/managers/WaveManager.js:57-61` (timer indipendente)
  - `src/main.js:2075-2150` (`startIntermission`)

## Proposed Fix

L'intermission timer dovrebbe essere **pausato** mentre la schermata modificatore è visibile, oppure la schermata modificatore dovrebbe essere mostrata PRIMA di avviare l'intermission e il timer dovrebbe partire solo dopo la scelta.

## Notes
- Lo stesso pattern si applica anche al mini-boss defeat in `src/managers/MiniBossManager.js:349-366`
- La guardia `window.Game.gameState` in `ModifierChoiceScreen._onKeyDown` non funziona — gameState non è mai stato esposto come proprietà di `window.Game`
